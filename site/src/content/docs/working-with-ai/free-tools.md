המסמך הזה נועד למי שאין לו מנוי AI בתשלום, ורוצה הקמה אג'נטית עובדת. זהו נספח ספציפי-לכלי ל[מסמך העקרונות](/alpha-onboarding/working-with-ai/principles/) — העקרונות שם הם החוזה; כאן רק הטכניקה.

> **שימו לב:** הצעות ה-free tier משתנות במהירות. המסמך מונה את הכלים ואת מה שהם כללו **נכון לזמן הכתיבה**, ונותן את צורת ההקמה היציבה. אל תסמכו על מספרי מכסה ספציפיים — בדקו את המגבלות העדכניות באתר של כל כלי לפני שאתם מסתמכים עליהן.

## העיקרון

כל כלי שיודע (א) לקרוא את הריפו שלכם, (ב) להציע diffs, ו-(ג) להריץ פקודות באישורכם — תומך בכל התוכנית. מסמך העקרונות הוא החוזה; הכלי עצמו ניתן להחלפה. בחרו אחת מהאופציות למטה.

## אופציה א' — Gemini CLI

כלי שורת-פקודה אג'נטי עם free tier מבוסס חשבון Google אישי.

- **התקנה:** דרך npm (`npm install -g @google/gemini-cli`) או winget על Windows. בדקו את הוראות ההתקנה העדכניות בעמוד הפרויקט הרשמי.
- **התחברות:** עם חשבון Google אישי (ה-free tier).
- **קובץ קונטקסט:** `GEMINI.md` בשורש הריפו (אותו רעיון כמו `CLAUDE.md`).
- **זרימה בסיסית:** שאלו ← קראו את ה-diff המוצע ← אשרו. לעולם אל תאשרו diff שלא קראתם.
- **על Windows/PowerShell:** הריצו מתוך טרמינל מאותחל; אם פקודות shell בסגנון POSIX נכשלות, הריצו אותן דרך Git Bash.

## אופציה ב' — GitHub Copilot (free tier)

- **מה כלול ב-free tier:** השלמות מוגבלות + chat, בתוך VS Code או Android Studio (דרך הפלאגין).
- **איפה זה מתאים:** מצוין ל-Explainer ול-Navigator — "תסביר את הקטע הזה", "איפה מוגדר X".
- **איפה זה חלש יותר:** משימות אג'נטיות מלאות (Generator על פני כמה קבצים) פחות חזקות ב-free tier. לשם כך שקלו אופציה א'.

## אופציה ג' — כל chat AI (מסלול הגיבוי)

כל chat חינמי (ChatGPT / Claude / Gemini ב-web) מכסה את מצב **Explainer** במלואו: הדביקו קוד, שאלו שאלות, הבינו. איטי יותר ל-Navigator (אין לו גישה ישירה לריפו), ואין Generator. לגמרי מספיק לשלבים 0–2, שבהם ממילא אתם בעיקר לומדים ומסבירים.

## הגדרת קובץ קונטקסט

ביום הראשון של פרויקט הגמר, צרו את קובץ הקונטקסט בריפו של פרויקט הגמר. אל תעתיקו אותו מאי-שם — **כתבו אותו**, וזרעו אותו במוסכמות שאתם לומדים. שקפו את החלקים הרלוונטיים מתוך ה-`CLAUDE.md` של alpha-mobile:

- כללי MVVM (interface + `Impl`, `StateFlow<ScreenState>` immutable).
- מוסכמות שמות (PascalCase למחלקות, סיומת `Impl`, `_` ל-backing state).
- האיסורים: אין `MutableStateFlow` חשוף, אין ניווט מבוסס-מחרוזות.

עצם הכתיבה של הקובץ היא תרגיל למידה — אתם מנסחים לעצמכם את מה שאתם קולטים.

## תרגיל

עם הכלי שבחרתם, הריצו את התרגיל מ[מסמך העקרונות, סעיף "תרגיל"](/alpha-onboarding/working-with-ai/principles/#תרגיל) — אם עוד לא עשיתם אותו — הפעם מקצה לקצה דרך הכלי הזה. **הגדרת סיום:** הכלי הסביר לכם את `SearchViewModel.kt`, ואתם מצאתם ורשמתם לפחות אי-דיוק אחד.
