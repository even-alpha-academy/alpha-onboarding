המטרה של היום הזה היא פשוטה: עד סוף יום העבודה הראשון יהיו לכם כל הכלים מותקנים, שני הריפואים משוכפלים, בילד ה-mock של alpha-mobile רץ על מכשיר היעד שלכם, וה-PR הראשון פתוח. זה מסמך מסוג צ'קליסט — עברו עליו לפי הסדר, וסמנו תוך כדי. אם משהו נתקע יותר מ-15 דקות, זה הזמן לקרוא למנטור (ראו את כלל שלושת השלבים ב[מסמך הפתיחה](/alpha-onboarding/welcome/)).

## 1. מה צריך מהצוות (לפני שמתחילים)

לפני היום הראשון, המנטור כבר אמור להכין עבורכם (ראו [מדריך המנטור](/alpha-onboarding/mentor/guide/)):

- הרשאות לארגון ה-GitHub **even-alpha** (גישת קריאה ל-`alpha-mobile` ולריפו החפיפה).
- מחשב עם הרשאות אדמין (נדרש להתקנות ול-emulator/drivers).
- אופציונלי: מכשיר אנדרואיד פיזי לבדיקות (במיוחד לשלב המפה בפרויקט הגמר).

אם אחד מאלה חסר — זה הנושא הראשון לסנכרון הבוקר, לא חסם שתפתרו לבד.

## 2. התקנות

התקינו את הכלים הבאים. אחרי כל התקנה, ודאו עם `--version` (או דרך ה-UI) שהיא תפסה.

| כלי | מה להתקין | הערה |
|---|---|---|
| Android Studio | הגרסה היציבה האחרונה | דרך ה-SDK Manager התקינו את ה-SDK Platforms לטווח 28–35 |
| JDK 21 | Java 21 | Android Studio מגיע עם JDK מובנה, אבל ה-Gradle toolchain כאן דורש 21 — ודאו שהוא זמין |
| git | הגרסה האחרונה | על Windows: התקינו גם Git Bash, חלק מהסקריפטים הם `.sh` |
| Node.js | גרסת LTS | נדרש ל-backend ב-Nest בשלב 1 |
| מנהל חבילות | npm (מגיע עם Node) או pnpm/yarn | לבחירתכם |
| MongoDB | MongoDB Community Server, התקנה native | **לא** דרך Docker — נורמת הצוות היא התקנות native |

<Aside type="note" title="למה Mongo ו-Node כבר עכשיו?">
בשלב 1 תבנו את ה-backend של פרויקט הגמר (ב-Nest, מול MongoDB). נוח שהדרישות מותקנות מראש כדי שלא תיתקעו על תשתית באמצע התרגיל.
</Aside>

## 3. בחירת מכשיר היעד

מכאן והלאה כל המסמכים מדברים על **"מכשיר היעד"** — המקום שבו תריצו את האפליקציה. יש שני מסלולים, ושניהם תקפים לכל התוכנית. בחרו אחד (יחד עם המנטור); אפשר להחליף בהמשך בלי לשבור כלום.

<Tabs>
  <TabItem label="אמולטור">
    - **הקמה:** צרו AVD מסוג Pixel, API 34+, עם האצת חומרה (hardware acceleration) דלוקה.
    - **יתרונות:** לא צריך חומרה; קל לצלם מסך; אפשר כמה מופעים.
    - **חסרונות:** אמולציית GPU לפעמים מתקוטטת עם ה-SDK של המפה התלת-ממדית (כדאי לדעת כבר עכשיו).
    - בדקו שהאצת החומרה פעילה. על Windows זה WHPX/Hypervisor (או HAXM בגרסאות ישנות). בלי זה האמולטור יזחל.
  </TabItem>
  <TabItem label="מכשיר פיזי">
    - **הקמה:** Developer options + USB debugging; על Windows ודאו driver מתאים.
    - **יתרונות:** GPU אמיתי — שלב המפה התלת-ממדית רץ בדרך כלל חלק יותר; מגע אמיתי.
    - **חסרונות:** חיכוך של כבל/drivers; צריך מכשיר (המנטור עשוי לספק אחד).
    - הפעילו Developer options (הקשה 7 פעמים על "Build number"), הדליקו USB debugging, חברו בכבל, ואשרו את ה-prompt. בדיקת שפיות: `adb devices` צריך להציג את המכשיר כ-`device` (לא `unauthorized`).
  </TabItem>
</Tabs>

**המלצה:** שני המסלולים עובדים לכל התוכנית. אם יש מכשיר פיזי זמין — העדיפו אותו החל משלב המפה (שלב 2 בפרויקט הגמר) והלאה. מעבר בהמשך זול; שום דבר בתוכנית לא קשור למסלול מסוים.

## 4. שכפול הריפואים

א. **ריפו החפיפה** (זה שאתם קוראים בו עכשיו) — שכפלו אותו מקומית.

ב. **alpha-mobile** — שכפלו מהארגון `even-alpha`, ואז הריצו פעם אחת את סקריפט ההקמה:

```bash
./scripts/setup-clone.sh
```

הסקריפט מגדיר `core.hooksPath=.githooks` ו-`core.autocrlf=false`, ומפעיל את ה-git hooks. הוא **חובה** ומריצים אותו פעם אחת לכל clone. מה שהוא בעצם עושה: מתקין hook מסוג `pre-push` שסורק תוכן רגיש לפני כל push. כרגע פשוט הריצו אותו וסמכו עליו — הסיפור המלא של מודל שתי הרשתות והדיסציפלינת-סודות מחכה ב[סודות ורשתות](/alpha-onboarding/alpha/secrets-networks/).

## 5. הבילד הראשון

מתוך תיקיית `alpha-mobile`, הריצו:

```bash
./gradlew assembleDevDebug
```

מה בדיוק בניתם? את ה-flavor החיצוני/mock של האפליקציה. המפתח `alpha.connectivity` ריק ב-`gradle.properties`, ולכן הבילד מתקמפל מול `src/external` — קוד מדומה (mocks), assets-stub, בלי keystores ובלי שרתים אמיתיים. זו בדיוק הסביבה שמיועדת לכם בצד הפתוח, ואין בה שום חשיפה.

התקינו את ה-APK על מכשיר היעד, התחברו דרך זרימת ה-mock, והסתכלו על המפה. **התוצאה המצופה: אפליקציה שרצה.**

<Aside type="tip" title="פתרון תקלות נפוצות">
- *Gradle JDK mismatch:* ב-Android Studio → Settings → Build Tools → Gradle, ודאו שה-Gradle JDK הוא 21.
- *long paths על Windows:* אם הבילד נופל על נתיב ארוך מדי, הפעילו תמיכה ב-long paths (מדיניות `LongPathsEnabled`) ושכפלו לנתיב קצר (למשל `C:\dev\alpha-mobile`).
- *אם אתם על אמולטור:* בעיות רינדור → ודאו שהאצת החומרה דלוקה ושה-AVD על API 34+.
- *אם אתם על מכשיר פיזי:* `adb devices` מציג `unauthorized` → אשרו את חלון ה-USB debugging; לא רואים את המכשיר בכלל → בדקו את ה-driver / החליפו כבל-data.
</Aside>

## 6. ה-PR הראשון שלך (היום!)

המשימה: מצאו משהו בריפו **הזה** ששווה לתקן — טעות כתיב, משפט לא ברור, לינק שבור, או צעד התקנה חסר שנתקלתם בו עכשיו. צרו branch, תקנו, ופתחו PR לריפו החפיפה. המנטור יעבור עליו עוד היום.

למה זה התרגיל הראשון? כי ביום הראשון אתם כבר מריצים את הלולאה המלאה — branch, PR, review, merge — על משהו אמיתי וקטן. ובונוס: התיעוד משתפר עם כל חניך.
