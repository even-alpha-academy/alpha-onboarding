זהו תרגיל היציאה של שלב היסודות (מבחן מדרגה 1): מסך אחד עצמאי שמוכיח שכל הסטאק נחת — Kotlin, coroutines, Compose, MVVM ו-Koin יחד. החופף יסקור אותו **כמו MR אמיתי** (לפי המחוון של החופף). זו נקודת המדידה לפני שמתחילים את פרויקט הגמר.

## המשימה

בנו mini-app (מסך + פירוט) ב**ריפו התרגיל** שהחופף הקים לכם (ראו [תהליך ה-GitHub](/alpha-onboarding/setup/#ריפו-ייעודי-לכל-תרגיל)):

- **מסך "רשימת שטחי אימון"** — רשימת פריטי `TrainingArea` מ-repository **מזויף** (in-memory; `suspend fun` שמחזיר אחרי `delay(500)` — בלי רשת). המסך חייב להציג שלושה מצבים:
  - **loading** — בזמן ההמתנה.
  - **error** — ה-repo המזויף נכשל לפי דרישה (דגל שתדליקו כדי לבדוק את הענף).
  - **empty** — רשימה ריקה.
- **הקשה על פריט ← מסך פירוט** דרך ניווט typed (`sealed Route` + extension על `NavGraphBuilder`).
- **הארכיטקטורה חייבת לעקוב אחרי תבנית הצוות במדויק:**
  - ViewModel: `interface`+`Impl`, `StateFlow<ScreenState>`, `data class` אחד immutable, `_state` פרטי.
  - Repository: `interface`+`Impl`.
  - Koin: מודולים שמחווטים הכל; `koinViewModel` ב-route.
- **לפחות 2 unit tests ל-ViewModel** — happy path + error path — במבנה בסגנון `TestSetup` (`runTest`, repository ממוקק; MockK ישירות זה בסדר, `declareMock` אופציונלי).

## מה מותר ומה לא (AI)

- **Explainer** — מותר. **Navigator** — מותר.
- **Generator — אסור.** אתם כותבים כל שורה.

למה? כי זו **נקודת המדידה**. מבחן המדרגה בודק אם אתם, בעצמכם, שולטים בתבנית. אחרי שתעברו אותו, החופף ישקול לפתוח את Generator ל-Kotlin (ראו את [סולם המצבים](/alpha-onboarding/working-with-ai/principles/#סולם-המצבים)).

## איך מגישים

צרו branch בריפו התרגיל ופתחו MR עם תיוג `gate-1`. החופף סוקר תוך יום, בדיוק כמו MR אמיתי — כולל הערות שתצטרכו לתקן.

## קריטריוני מעבר

(תואם את מחוון מבחן מדרגה 1 אצל החופף.)

- [ ] מתקמפל ורץ על מכשיר היעד.
- [ ] שלושת מצבי הרשימה (loading / error / empty) ניתנים להגעה.
- [ ] **אפס** הפרות של תבנית הארכיטקטורה — אף הפרה של מוסכמות הצוות (§5 Conventions / §2 ב-`CLAUDE.md`): אין `MutableStateFlow` חשוף, אין ניווט מבוסס-מחרוזות, יש interface+Impl.
- [ ] הטסטים עוברים **ובאמת בודקים מעברי state** (לא רק "לא קרס").
- [ ] אתם עונים בסקירה על שתי שאלות "למה" — למשל: למה `StateFlow` ולא `LiveData` כאן, ולמה `factory` ולא `single` ל-repository.

## נפלת על משהו?

זה נורמלי. מבחן המדרגה מודד **מוכנות, לא ערך**. החופף מחליט: מעבר / יום ריפוי ממוקד / התחלת פרויקט הגמר תוך ריפוי במקביל. אף אחת מהאפשרויות אינה כישלון — כולן דרכים קדימה.
