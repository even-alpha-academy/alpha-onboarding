זהו תרגיל היציאה של שלב היסודות (שער 1): מסך אחד עצמאי שמוכיח שכל הסטאק נחת — Kotlin, coroutines, Compose, MVVM ו-Koin יחד. המנטור יסקור אותו **כמו MR אמיתי** (המחוון נמצא ב[מחווני ההערכה של המנטור](/alpha-onboarding/mentor/rubrics/)). זו נקודת המדידה לפני שמתחילים את פרויקט הגמר.

## המשימה

בנו mini-app (מסך + פירוט) בפרויקט חדש בשם `gate1/`:

- **מסך "רשימת שטחי אימון"** — רשימת פריטי `TrainingArea` מ-repository **מזויף** (in-memory; `suspend fun` שמחזיר אחרי `delay(500)` — בלי רשת). המסך חייב להציג שלושה מצבים:
  - **loading** — בזמן ההמתנה.
  - **error** — ה-repo המזויף נכשל לפי דרישה (דגל שתדליקו כדי לבדוק את הענף).
  - **empty** — רשימה ריקה.
- **הקשה על פריט → מסך פירוט** דרך ניווט typed (`sealed Route` + extension על `NavGraphBuilder`).
- **הארכיטקטורה חייבת לעקוב אחרי תבנית הצוות במדויק:**
  - ViewModel: `interface`+`Impl`, `StateFlow<ScreenState>`, `data class` אחד immutable, `_state` פרטי.
  - Repository: `interface`+`Impl`.
  - Koin: מודולים שמחווטים הכל; `koinViewModel` ב-route.
- **לפחות 2 unit tests ל-ViewModel** — happy path + error path — במבנה בסגנון `TestSetup` (`runTest`, repository ממוקק; MockK ישירות זה בסדר, `declareMock` אופציונלי).

## מה מותר ומה לא (AI)

- **Explainer** — מותר. **Navigator** — מותר.
- **Generator — אסור.** אתם כותבים כל שורה.

למה? כי זו **נקודת המדידה**. השער בודק אם אתם, בעצמכם, שולטים בתבנית. אחרי שתעברו אותו, המנטור ישקול לפתוח את Generator ל-Kotlin (ראו את [סולם המצבים](/alpha-onboarding/working-with-ai/principles/#סולם-המצבים)).

## איך מגישים

צרו branch ופתחו MR (בריפו פרויקט הגמר, או בריפו scratch) עם תיוג `gate-1`. המנטור סוקר תוך יום, בדיוק כמו MR אמיתי — כולל הערות שתצטרכו לתקן.

## קריטריוני מעבר

(תואם את מחוון שער 1 אצל המנטור.)

- [ ] מתקמפל ורץ על מכשיר היעד.
- [ ] שלושת מצבי הרשימה (loading / error / empty) ניתנים להגעה.
- [ ] **אפס** הפרות של תבנית הארכיטקטורה — אף תיקון מהסוגים המופיעים ב"Common Pitfalls" של `CLAUDE.md` (אין `MutableStateFlow` חשוף, אין ניווט מבוסס-מחרוזות, יש interface+Impl).
- [ ] הטסטים עוברים **ובאמת בודקים מעברי state** (לא רק "לא קרס").
- [ ] אתם עונים בסקירה על שתי שאלות "למה" — למשל: למה `StateFlow` ולא `LiveData` כאן, ולמה `factory` ולא `single` ל-repository.

## נפלת על משהו?

זה נורמלי. השער מודד **מוכנות, לא ערך**. המנטור מחליט: מעבר / יום ריפוי ממוקד / התחלת פרויקט הגמר תוך ריפוי במקביל. אף אחת מהאפשרויות אינה כישלון — כולן דרכים קדימה.
