זהו השלב האחרון של פרויקט הגמר (ימים 19–22), והוא הכי דומה לטיקט אלפא אמיתי: עבודת feature מלאה (full-stack) במהירות. הוא נגמר במבחן מדרגה 2.

## המשימה

שלושה מסכים:

1. **"תעד תרגול"** — כפתור ב-bottom sheet של השטח → טופס (סוג, משך, דירוג 1–5, הערות חופשי; התאריך מוגדר כברירת מחדל ל"עכשיו") → `POST /exercises` → משוב הצלחה + חזרה.
2. **"היסטוריית תרגולים באזור"** — כפתור ב-sheet → רשימה לאותו שטח (ממוינת לפי תאריך יורד), הקשה → פירוט.
3. **"התרגולים שלי"** — כניסה ממסך ההגדרות → כל התרגולים שלי על פני כל השטחים.

## דרישות

- **Validation בטופס** (משך > 0, דירוג 1–5) **ברמת ה-ViewModel**, עם מצבי שגיאת-שדה.
- **שימוש חוזר**: composable אחד משותף לפריט-רשימה, על פני מסכים ב' ו-ג'. השימוש החוזר **נבדק**.
- **ה-repository מתרחב נקי** — ה-interceptor משלב 1 נושא את ה-token, אין צורך לחווט token לכל קריאה.
- **ניווט דרך Routes typed עם ארגומנטים** (`areaId`). גשר: routes פרמטריים ב-`Route.kt` של alpha-mobile — `SketchEditor` מקבל ארגומנטים, דוגמה אמיתית.

## טסטים

עכשיו לא ניתנים למשא ומתן, ורחבים יותר:

- טסטי ViewModel למטריצת ה-validation (משך/דירוג חוקיים ולא-חוקיים) + טעינת רשימה (loading/error).
- טסט repository אחד עם API ממוקק.
- מבנה בסגנון `TestSetup` (`@Nested`, `runTest`).

ה-MR הזה נסקר **טסטים-תחילה** — "הטסטים נקראים לפני הקוד".

## מדיניות AI

Navigator בחופשיות. אם Generator נפתח — זהו השלב הנכון לתרגל אותו על קוד feature אמיתי, עם [צ'קליסט הסקירה](/alpha-onboarding/working-with-ai/principles/#לסקור-קוד-שלא-כתבת) מיושם **גלוי** בתיאור ה-MR: "מה ה-AI כתב, מה תיקנתי ולמה" — פסקה אחת, שבונה את הרגל הרפלקציה.
